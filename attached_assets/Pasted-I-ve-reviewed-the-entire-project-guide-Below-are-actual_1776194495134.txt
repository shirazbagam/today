I’ve reviewed the entire project guide. Below are actual mistakes, inconsistencies, and design issues that could cause problems during development.

1. Database Schema – Critical Design Flaw

Mistake:
users, riders, and vendors are separate tables with no relationship to a common auth table.

A person cannot be both a customer and a rider/vendor.

Authentication (login) would need to check three different tables.

Wallet, notifications, and addresses would be duplicated across tables.

Fix:
Use a single users table with a role column (customer, rider, vendor, admin).
Then create rider_profiles, vendor_profiles linked via user_id.

ts

users (id, phone, password, role, name, wallet_balance, created_at) rider_profiles (user_id, vehicle_type, is_online, current_location, total_earnings) vendor_profiles (user_id, store_name, location, balance)

2. Missing Tables / Columns

No customer_wallet or user_wallet – only wallet_transactions.
Add: users.wallet_balance or a separate wallets table.

No refresh_tokens table for secure auth sessions.

No order_status_history – you can’t track when an order moved from “confirmed” to “preparing”.

riders table has wallet but no completed_orders_count or rating.
